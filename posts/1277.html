单例模式（Singleton）

六月 11, 2016
  • 设计模式

单例模式（Singleton），保证一个类仅有一个实例，并提供一个访问它的全局访问点。
单例模式有以下特点：

  • 单例类只能有一个实例。
  • 单例类必须自己创建自己的唯一的实例。
  • 单例类必须能给所有其他对象提供这一实例。

代码实现

饿汉式单例类

在单例类被加载时就将自己实例化

package com.qinshaoxuan.singleton;

public class EagerSingleton {

    private static final EagerSingleton instance = new EagerSingleton();

    /**
     * 私有的构造方法,保证外界无法直接实例化
     */
    private EagerSingleton(){}

    /**
     * 静态工厂方法
     * @return 单例类的唯一实例
     */
    public static EagerSingleton getInstance(){
        return instance;
    }
}

懒汉式单例类

在单例类第一次被引用时将自己实例化

package com.qinshaoxuan.singleton;

public class LazySingleton {

    private static LazySingleton instance = null;

    /**
     * 私有的构造方法,保证外界无法直接实例化
     */
    private LazySingleton() {}

    /**
     * 静态工厂方法
     * @return 单例类的唯一实例
     */
    synchronized public static LazySingleton getInstance(){
        if(instance == null){
            instance = new LazySingleton();
        }
        return instance;
    }
}

登记式单例类

登记式单例类是为了克服饿汉式和懒汉式单例类均不可继承的缺点而设计的。

这里的实现为父类饿汉式，子类懒汉式，且子类只能是懒汉式。

由于子类必须允许父类以构造方法调用产生实例，因此，它的构造方法必须是公开的。这样一来就等于允许了以这样的方式产生实例而不在父类的登记中，这是登记式单例类的一个缺点。

由于父类的实例必须存在才可能有子类的实例，这在有些情况下是一个浪费，这是登记式单例类的另一个缺点。

 

RegSingleton类

package com.qinshaoxuan.singleton;

import java.util.HashMap;

public class RegSingleton {

static private HashMap registry = new HashMap();

static {
    RegSingleton instence = new RegSingleton();
    registry.put(instence.getClass().getName(), instence);
}

/**
 * 保护的构造方法
 */
protected RegSingleton() {}

/**
 * 静态工厂方法
 * @param name 单例类的类名
 * @return 单例类的唯一实例
 */
static public RegSingleton getInstence(String name){
    if(name == null){
        name = "com.qinshaoxuan.singleton.RegSingleton";
    }
    if(registry.get(name)==null){
        try{
            registry.put(name, Class.forName(name).newInstance());
        }
        catch(Exception e){
            System.out.println("Error happened.");
        }
    }
    return (RegSingleton)(registry.get(name));
}

}

RegSingletonChild类

package com.qinshaoxuan.singleton;

public class RegSingletonChild extends RegSingleton {

/**
 * 构造方法
 */
public RegSingletonChild() {}

/**
 * 静态工厂方法
 * @return 单例类子类的唯一实例
 */
static public RegSingletonChild getIntance(){
    return (RegSingletonChild)RegSingleton.getInstence("com.qinshaoxuan.singleton.RegSingletonChild");
}

}

 

优点

在系统内存中只存在一个对象，因此可以节约系统资源，对于一些需要频繁创建和销毁的对象使用单例模式无疑可以提高系统的性能。

 

使用场合

在一个系统要求一个类只有一个实例时才应当使用单例模式。反过来说，如果一个类可以有几个实例共存，那么就没有必要使用单例类。
